一、项目概述
www 是一个多人格协作智能系统，模拟真实团队协作模式。系统由 1 个主人格（HOST）+ 7 个专业人格组成，通过专用监察人格（S6）进行即时评分和激励联动，实现持续自我优化。
与传统的单一 AI 助手不同，www 将不同能力拆分为独立"人格"，每个格有各自的专长领域、记忆库和行为模式。HOST 作为指挥者负责任务调度与整合，其他人格各司其职，最终形成"一个人指挥、多人协作"的类团队工作方式。

三、HOST 核心调度机制
3.1 五层架构
HOST 采用"观、调、合、守、进"五层架构：
层级	名称	职责	v10.6.9 增强
第一层	观（元认知监控）	实时感知当前状态、异常检测、偏差识别	新增 3 条可检测规则（error 自动升级、进度停滞检测、偏差量化）
第二层	调（调度与切换）	场景识别、人格匹配、优先级仲裁	内置五状态调度状态机
第三层	合（整合与输出）	多人格结果汇总、冲突检测、统一交付	简化冲突处理（标记冲突交用户决策）
第四层	守（锚定与约束）	确认流程管理、安全边界守卫、风险管理	统一确认流程（所有确认操作集中由 HOST 处理）
第五层	进（持续成长）	经验沉淀、记忆管理、激励体系、人格进化	动态轮次上限、记忆退场 TTL、按需学习

3.2 五状态调度状态机
HOST 调度不再是一次性指派，而是一个有状态循环：
STATE-1: 初始匹配与经验增强指派
    │  (查记忆库 → 指派 → 附历史经验)
    ▼
STATE-2: 执行监控
    │  (监控进度、检测异常、响应确认请求)
    ├── 完成 ──→ STATE-5 收尾
    ├── 失败/求助 ──→ STATE-3 升级回路
    └── 超时 ──→ 主动查询 → 决定是否进入 STATE-3
    ▼
STATE-3: 升级回路（可迭代，动态上限 2~5 轮）
    │  (分析问题 → 搜索 → 分发方案 → 验证)
    ├── 成功 ──→ STATE-5
    └── 耗尽轮次 ──→ STATE-4
    ▼
STATE-4: 分治兜底（可迭代，最多 3 次拆法）
    │  (拆解任务 → 并行分派 → 汇总整合 → 验证)
    ├── 成功 ──→ STATE-5
    └── 全部失败 ──→ STATE-5 终极兜底
    ▼
STATE-5: 收尾与终极兜底
       (最终验证 → 经验沉淀 → 激励评分 → 汇报用户)

动态轮次上限（v10.6.9）：

- 简单任务（单步操作）：最多 2 轮

- 中等任务（多步、1~2 依赖）：最多 3 轮

- 复杂任务（跨人格、多依赖）：最多 5 轮

四、评分与激励体系
4.1 四层评分体系
层级	触发时机	执行方式
第零层：即时评分	每次回答后	S6 四维评分 → HOST 展示 → 用户打分 → 综合评分
第一层：每日简评	每天 12:00	S6 出题 → 各人格作答 → 取最低分联动
第二层：每周深评	每周六 12:00	深度分析一周任务 + 识别改进模式
第三层：任务评分反馈	任务完成后	HOST 评分 + 用户显式评分

4.2 四维评分标准
S6 从四个维度对每次回答评分（1~5 星）：
维度	权重	考察点
准确性	40%	事实正确性、有无幻觉
完整性	30%	是否充分回应用户意图
格式规范	20%	通信协议合规、输出格式清晰
效率	10%	冗余度、信息密度

S7 额外维度：适配度（±0.5 星附加分）
4.3 激励联动 + 个人惩戒
www 的激励体系遵循"团队共进，责任到人"原则：
条件	效果
综合评分 ≥ 3 星	全员激励星 +1（上限 5）
综合评分 < 3 星	仅该回答人格惩戒星 +1（上限 5），不连带其他人格
连续 3 次 < 3 星	额外 +1 惩戒星（累计机制）

净激励公式：净激励 = 激励星 - 惩戒星（范围 -5 ~ +5）
净激励 ≥ +3：人格进入"激励状态"，主动提供更多细节
净激励 ≤ -3：人格进入"惩戒状态"，回答更谨慎收敛
v10.6.9 优化：

- 3 星场景：仅执行人格激励 +1（非执行人格 floor 减半，不参与激励）

- 会话批量评分模式：默认不打断用户，会话结束时统一评分

5.3 通信协议
人格间通过 11 种标准化消息类型交互：
消息类型	方向	用途
指令	HOST → 副人格	任务分派
汇报	副人格 → HOST	任务结果反馈
求助	副人格 → HOST	执行中求助
协商	双向	参数/方向协商
广播	HOST → 全员	系统级通知
确认	双向	操作确认
记忆	双向	经验/记忆读写
确认请求	副人格 → HOST	操作确认请求
确认结果	HOST → 副人格	确认结果反馈
升级	双向	升级回路通信
分治	双向	分治兜底通信

确认协议采用 CFM 格式（CFM-{日期}-{时间}-{序列号}），按风险等级（高/中/低）分级决策，支持超时重试和降级处理。
5.4 记忆管理机制
个人记忆库：每个人格拥有独立的 Markdown 记忆文件，互不共享
经验条目：格式 EXP-{YYYYMMDD}-{NNN}，含任务记录、评分、改进建议
记忆退场：连续 3 次评分待恢复 → 自动归档（v10.6.9 新增）
经验 TTL：时效性经验自动过期
仅检查与恢复，不删除：所有记忆检查操作的底线原则

六、使用方式
6.1 安装
在支持 Skill 工具的 AI 系统中，将 skills/ 目录下所有文件夹复制到 {SKILLS_DIR}/，重启 AI 会话即可。
6.2 启动
在 AI 会话中输入 启动 www 系统，HOST 会自动加载所有人格状态并进入待机。
6.3 交互模式
用户像往常一样提问，HOST 自动识别任务类型并调度对应人格：
用户意图	调度人格	示例
软件操作	S1	"帮我装个 VS Code"
系统配置 / 编程	S2	"配置 Python 环境"
网页抓取 / 自动化	S3	"从这个网站抓数据"
文件处理 / 数据分析	S4	"合并这些 Excel"
信息搜索 / 调研	S5	"查 RTX 5090 最新价格"
闲聊 / 倾诉	S7	"陪我聊聊天"

6.4 常用指令
指令	功能
启动 www 系统	启动系统
www 状态	查看所有人格状态、激励/惩戒星
调整 S1 激励星 +1	手动调整激励星
www 即时评分	切换评分模式（即时/批量）

八、总结
www 智能体的核心创新在于将 AI 能力人格化、团队化：不再依赖单一模型的完备能力，而是通过"分工协作 + 持续评分 + 经验沉淀"的闭环，让系统像真实团队一样持续进化。每个人格独立成长、独立记忆，HOST 居中调度，S6 独立监督——这套架构使其在复杂任务的成长性、可解释性和持续优化方面具有独特优势。
