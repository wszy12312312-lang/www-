## 项目愿景

www 的目标是打造一个**可自主操控电脑、无需人工干预、具备持续成长性**的多人格智能系统。系统将 AI 能力人格化、团队化——不再依赖单一模型，而是通过"分工协作 + 持续评分 + 经验沉淀"的闭环，让系统像真实团队一样持续进化。

> **如果有人能帮忙改进，非常欢迎！** 初版由非专业开发者构建，技术有限但愿景明确——让 HOST 能自主增删人格、系统完全自动化运行。

## HOST 调度机制

### 五层架构

层级 | 名称 | 职责 | v10.6.9 增强
:---:|:---:|:---|:---
1 | **观**（元认知监控） | 实时感知状态、异常检测、偏差识别 | 新增 3 条检测规则
2 | **调**（调度与切换） | 场景识别、人格匹配、优先级仲裁 | 五状态调度状态机
3 | **合**（整合与输出） | 多人格结果汇总、冲突检测、统一交付 | 标记冲突交用户决策
4 | **守**（锚定与约束） | 确认流程、安全边界、风险管理 | 统一确认流程
5 | **进**（持续成长） | 经验沉淀、记忆管理、激励体系 | 动态轮次上限、记忆退场 TTL

### 五状态调度状态机

```
STATE-1 初始匹配 → 查记忆库 → 指派 → 附历史经验
STATE-2 执行监控 → 监控进度 / 检测异常 / 响应确认
    ├── 完成 → STATE-5 收尾
    ├── 失败/求助 → STATE-3 升级回路
    └── 超时 → 主动查询 → 决定是否进入 STATE-3
STATE-3 升级回路（迭代，动态上限 2~5 轮）
    ├── 成功 → STATE-5
    └── 耗尽轮次 → STATE-4
STATE-4 分治兜底（最多 3 次拆法）
    ├── 成功 → STATE-5
    └── 全部失败 → STATE-5 终极兜底
STATE-5 收尾 → 验证 → 经验沉淀 → 激励评分 → 汇报用户
```

动态轮次上限：简单任务 ≤ 2 轮 | 中等任务 ≤ 3 轮 | 复杂任务 ≤ 5 轮

## 评分与激励体系

### 四层评分

层级 | 触发时机 | 执行方式
:---:|:---|:---
第零层 | 每次回答后 | S6 四维评分 → HOST 展示 → 用户打分 → 综合
第一层 | 每天 12:00 | S6 出题 → 各人格作答 → 取最低分联动
第二层 | 每周六 12:00 | 深度分析一周任务 + 识别改进模式
第三层 | 任务完成后 | HOST 评分 + 用户显式评分

### 四维评分标准

维度 | 权重 | 考察点
:---:|:---:|:---
准确性 | 40% | 事实正确性、有无幻觉
完整性 | 30% | 是否充分回应用户意图
格式规范 | 20% | 通信协议合规、输出格式清晰
效率 | 10% | 冗余度、信息密度
适配度* | ±0.5⭐ | S6 额外附加分

### 激励联动 + 个人惩戒

> **团队共进，责任到人**

条件 | 效果
:---|:---
评分 ≥ 3⭐ | 仅执行人格激励星 +1（上限 5）
评分 < 3⭐ | 仅该回答人格惩戒星 +1（上限 5），不连带其他人格
连续 3 次 < 3⭐ | 额外 +1 惩戒星（累计机制）

净激励 = 激励星 - 惩戒星（-5 ~ +5）

- ≥ +3 → **激励状态**，主动提供更多细节
- ≤ -3 → **惩戒状态**，回答更谨慎收敛

v10.6.9：3⭐ 场景仅执行人格激励；会话批量评分模式（不打断用户）

## 技术架构

层面 | 技术实现
:---|:---
人格定义 | Markdown + YAML frontmatter（SKILL.md）
注册管理 | JSON 注册表（www_personas_registry.json）
记忆持久化 | Markdown 文件（个人记忆库 + 任务日志）
通信协议 | 11 种标准化消息类型
确认管理 | CFM 格式 + 风险分级 + 超时重试
经验沉淀 | 记忆退场 TTL + 归档机制
部署方式 | Skill 系统 + 文件系统
运行引擎 | Ollama 本地模型

### 通信协议（11 种消息类型）

类型 | 方向 | 用途
:---:|:---:|:---
指令 | HOST → 副人格 | 任务分派
汇报 | 副人格 → HOST | 结果反馈
求助 | 副人格 → HOST | 执行中求助
协商 | 双向 | 参数/方向协商
广播 | HOST → 全员 | 系级通知
确认 | 双向 | 操作确认
记忆 | 双向 | 经验/记忆读写
确认请求 | 副人格 → HOST | 请求确认
确认结果 | HOST → 副人格 | 确认反馈
升级 | 双向 | 升级回路通信
分 | 双向 | 分治兜底通信

确认协议格式：`CFM-{日期}-{时间}-{序列号}`，风险等级（高/中/低），超时重试 + 降级处理。

### 记忆管理

- 每个人格拥有独立 Markdown 记忆文件，互不共享
- 经验条目格式：`EXP-{YYYYMMDD}-{NNN}`，含任务记录、评分、改进建议
- 记忆退场：连续 3 次评分待恢复 → 自动归档（v10.6.9 新增）
- 经验 TTL：时效性经验自动过期
- 底线原则：仅检查与恢复，不删除

## 安装与使用

### 安装

1. 下载最新版本 zip 包（如 `www_v10.6.9_fixed.zip`）
2. 解压到本地目录
3. 确保 Ollama 已安装并运行（[Ollama 官网](https://ollama.ai)）
4. 将 `skills/` 目录下所有文件夹复制到 `{SKILLS_DIR}/`
5. 重启 AI 会话

### 启动

在 AI 会话中输入 **`启动 www 系统`**，HOST 自动加载所有人格状态并进入待机。

### 交互模式

用户意图 | 调度人格 | 示例
:---|:---:|:---
软件操作 | S1 | "帮我装个 VS Code"
系统配置 / 编程 | S2 | "配置 Python 环境"
网页抓取 / 自动化 | S3 | "从这个网站抓数据"
文件处理 / 数据分析 | S4 | "合并这些 Excel"
信息搜索 / 调研 | S5 | "查 RTX 5090 最新价格"
闲聊 / 倾诉 | S6 | "陪我聊聊天"

### 常用指令

| 指令 | 功能 |
|:---|:---|
| `启动 www 系统` | 启动系统 |
| `www 状态` | 查看所有人格状态、激励/惩戒星 |
| `调整 S1 励星 +1` | 手动调整激励星 |
| `www 即时评分` | 切换评分模式（即时/批量） |
